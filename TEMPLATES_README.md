# NDMO/NDI Templates and Forms Guide

## Overview

This directory contains standardized templates and forms for NDMO/NDI compliance evidence collection and reporting. All templates follow NDMO standards and can be filled out and attached as evidence in the compliance dashboard.

## Available Templates

### 1. Evidence Collection Templates
**Purpose:** Document evidence for each specification

**File Format:** `Evidence_{SpecID}.pdf`

**Contains:**
- Control and specification information
- Evidence type and description fields
- File upload tracking
- Compliance status checkboxes
- Notes and comments section
- Signature fields for approval

**Usage:**
1. Download the template for your specification
2. Fill in all required fields
3. Attach supporting documents
4. Get required signatures
5. Upload completed template as evidence

### 2. Compliance Report Templates
**Purpose:** Create comprehensive compliance reports for controls

**File Format:** `Compliance_Report_{ControlID}.pdf`

**Contains:**
- Report information and period
- Control details
- Compliance summary with scores
- Specification-level compliance details
- Findings and recommendations
- Action items
- Approval signatures

**Usage:**
1. Download the template for your control
2. Fill in compliance status for each specification
3. Document findings and recommendations
4. Complete action items section
5. Get management approval
6. Submit as part of annual compliance report

### 3. Audit Checklist Templates
**Purpose:** Conduct compliance audits for controls

**File Format:** `Audit_Checklist_{ControlID}.pdf`

**Contains:**
- Control information
- Audit date and auditor details
- Checklist items (Yes/No/N/A)
- Audit findings
- Recommendations
- Signature fields

**Usage:**
1. Download the checklist for your control
2. Complete all checklist items
3. Document findings
4. Provide recommendations
5. Get required signatures
6. Upload as audit evidence

## Template Structure

All templates include:

### Standard Sections:
- **Header:** NDMO/NDI branding and template type
- **Control Information:** Control ID, name, domain
- **Date Fields:** Creation date, review dates
- **Status Fields:** Compliance status, scores
- **Evidence Fields:** File names, locations, upload tracking
- **Notes Section:** Free text for additional information
- **Signature Section:** Prepared by, reviewed by, approved by

### Compliance Status Options:
- ☐ Compliant (C)
- ☐ Partially Compliant (PC)
- ☐ Non-Compliant (NC)

## How to Use Templates

### Step 1: Generate Template
1. Go to "Templates & Forms" page in the dashboard
2. Select the control or specification
3. Click "Generate Template" button
4. Download the PDF template

### Step 2: Fill Out Template
1. Open the PDF template
2. Fill in all required fields
3. Check appropriate status boxes
4. Attach supporting documents (if needed)
5. Add notes and comments
6. Get required signatures

### Step 3: Upload Completed Template
1. Go to "Documents & Evidence" page
2. Select the control/specification
3. Upload the completed template
4. The template will be stored as evidence

## Template Standards

All templates follow NDMO standards:

- **Format:** PDF (fillable forms)
- **Language:** English
- **Structure:** Standardized sections
- **Fields:** Required and optional fields clearly marked
- **Signatures:** Required approval workflow
- **Version Control:** Date stamps for tracking

## Best Practices

1. **Complete All Fields:** Fill in all required fields before submission
2. **Attach Supporting Documents:** Include relevant evidence files
3. **Get Approvals:** Ensure all required signatures are obtained
4. **Keep Copies:** Maintain copies of all completed templates
5. **Regular Updates:** Update templates when compliance status changes
6. **Version Control:** Use date stamps to track template versions

## Template Customization

Templates can be customized for specific needs:

- Add organization-specific fields
- Include additional checklist items
- Modify signature requirements
- Add domain-specific sections

## File Naming Convention

Templates follow this naming convention:
- Evidence: `Evidence_{DomainCode}_{ControlNum}_{SpecNum}.pdf`
- Reports: `Compliance_Report_{DomainCode}_{ControlNum}.pdf`
- Checklists: `Audit_Checklist_{DomainCode}_{ControlNum}.pdf`

Example:
- `Evidence_DG_1_1.pdf` - Evidence template for DG.1.1
- `Compliance_Report_DG_1.pdf` - Compliance report for DG.1
- `Audit_Checklist_DG_1.pdf` - Audit checklist for DG.1
